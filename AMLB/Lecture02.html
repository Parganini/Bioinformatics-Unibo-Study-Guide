Big Picture

Lecture 01 introduced linear regression as a model with parameters that must be learned from data. This lecture turns that idea into a concrete training procedure: gradient descent.

The key question is no longer just “what is the model?” but “how do we actually find good parameter values?”

The lecture first explains gradient descent in a general way, then applies it to linear regression, and finally extends the discussion from one feature to many features through multivariate notation.

Gradient Descent Intuition

Gradient descent is an iterative optimization algorithm used to minimize a cost function J(θ).

The lecture presents it with a physical intuition: imagine the cost function as a landscape of hills and valleys. You pick a starting point, look around, and move in the direction that takes you downhill most quickly.

Think physically: start somewhere on the surface, check the local slope, and repeatedly step in the direction of steepest descent.
Step-by-step descent
The cost function can be visualized as a surface over parameter values. Gradient descent moves across this surface toward lower cost.
Different paths
Different starting points can lead to different paths, which is why initialization matters in general optimization problems.

Gradient Descent Update Rule

The generic gradient descent algorithm repeatedly updates every parameter in order to reduce the cost function.

Repeat until convergence: $$ \theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta_0, \theta_1, \ldots, \theta_n) $$
Term Meaning
$$\theta_j$$ The parameter currently being updated
$$J(\theta_0, \theta_1, \ldots, \theta_n)$$ The cost function to minimize
$$\frac{\partial}{\partial \theta_j} J(\theta_0, \theta_1, \ldots, \theta_n)$$ The partial derivative showing the local slope
$$\alpha$$ The learning rate, controlling step size
$$\text{Repeat until convergence}$$ Apply the rule iteratively until updates become negligible

The derivative tells you how the cost changes locally, while the learning rate determines how aggressive each update should be.

Simultaneous Updates

One of the most important implementation details is that all parameters must be updated simultaneously at each iteration.

Incorrect approach

Update θ₀ first, then use that already-updated θ₀ while computing θ₁ in the same iteration.

This mixes old and new values and breaks the intended logic of the algorithm.

Correct approach

Compute all new parameter values from the same old parameter state first, then apply them together.

This preserves the true gradient descent step for that iteration.

Simultaneous updates are not a minor coding detail — they are part of the correct mathematical implementation of gradient descent.
Implementation detail
All parameters must be updated from the same previous state. Updating one parameter early changes the meaning of the next update.

What the Derivative Tells You

The derivative term tells you which direction reduces the cost.

Positive Slope
Move left

If the derivative is positive, the update subtracts a positive quantity, so the parameter decreases.

Negative Slope
Move right

If the derivative is negative, subtracting it increases the parameter value.

Zero Slope
Stop changing

If the derivative is zero, you are already at a stationary point and the update becomes zero.

So the algorithm does not need an extra rule saying “go left” or “go right.” The sign of the derivative already encodes the direction of motion.

Direction from slope
The sign of the derivative determines whether a parameter increases or decreases on the next step.

Learning Rate

The learning rate α controls the size of each step in gradient descent.

$$ \theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta_0, \theta_1, \ldots, \theta_n) $$
Too Small
Slow convergence

The algorithm moves in the correct direction, but progress is very slow and may waste computation.

Reasonable
Efficient convergence

The algorithm reduces the cost steadily and reaches a minimum in a practical number of steps.

Too Large
Overshooting

The algorithm may jump across the minimum, oscillate, fail to converge, or even diverge.

A useful point from the lecture is that even with a fixed learning rate, gradient descent can still converge, because the derivative naturally gets closer to zero near a minimum.

The learning rate is a hyperparameter: it is chosen by us to guide training, rather than learned as part of the model itself.
Learning rate comparison
From left to right: a too-small learning rate leads to slow convergence, a too-large learning rate causes overshooting or divergence, and a well-chosen value converges efficiently to the minimum.

Local vs Global Optima

In general optimization, gradient descent can end at different local minima depending on where it starts. That is a normal property of basic gradient descent.

General case

A complicated cost surface may contain many valleys. Starting from different initial points can lead to different local minima.

Linear regression case

The cost function for linear regression is convex, so there are no bad local minima — only one global optimum.

This is why gradient descent behaves especially well for linear regression: the optimization landscape is “bow shaped,” so the minimum is unique.
Optimization landscape
In general optimization problems, gradient descent can converge to different local minima depending on the starting point. In linear regression, the cost function is convex, so there is only one global minimum.

The caveat is that the learning rate still has to be chosen sensibly. Convexity helps, but an excessively large α can still break convergence.

Applying Gradient Descent to Linear Regression

Once the hypothesis and cost function are defined for linear regression, gradient descent becomes concrete: compute the derivatives of the cost function with respect to the parameters and plug them into the generic update rule.

$$h_{\theta}(x) = \theta_0 + \theta_1 x$$
$$J(\theta_0, \theta_1) = \frac{1}{2m} \sum_{i=1}^{m} (h_{\theta}(x^{(i)}) - y^{(i)})^2$$

The lecture emphasizes that what changes from one model to another is mainly the derivative term, because that depends on the chosen hypothesis.

Early training
At the beginning, the fitted line may still point in a poor direction and not match the data well.
Later training
As gradient descent continues, the line gradually rotates and shifts toward a better fit.

Batch Gradient Descent

The version presented here is called batch gradient descent.

“Batch” means that every gradient update uses the entire training set, not just a subset of examples.

In the update formulas, this appears through sums that run over all training examples from i = 1 to m. Other variants exist, but this lecture introduces the full-batch version first.

From Univariate to Multivariate Linear Regression

Real problems almost never use just one feature. Multivariate linear regression extends the same idea to many inputs.

Univariate

One feature is used to make a prediction.

$$h_{\theta}(x) = \theta_0 + \theta_1 x$$

Multivariate

Multiple features are used together to make a prediction.

$$h_{\theta}(x) = \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \ldots + \theta_n x_n$$

Notation

Symbol Meaning
$$m$$ Number of training examples (rows)
$$n$$ Number of features
$$x^{(i)}$$ Input features of the i-th training example
$$x_j^{(i)}$$ Value of feature j in example i
$$y^{(i)}$$ Target value of the i-th training example

This notation becomes necessary because once there are many features, each example is no longer a single number but a full feature vector.

Vectorized Notation

Writing the multivariate hypothesis term by term quickly becomes inconvenient. To simplify notation, the lecture introduces a “zero feature” defined by:

$$x_0 = 1$$

This lets the intercept term θ₀ be absorbed into the same vector structure as the other features.

$$x = [x_0, x_1, \ldots, x_n]^T$$
$$\theta = [\theta_0, \theta_1, \ldots, \theta_n]^T$$
$$h_{\theta}(x) = \theta^T x$$
Object Interpretation
$$x_0 = 1$$ Artificial feature added only for notation convenience
$$x$$ Feature vector for one example
$$\theta$$ Parameter vector to learn
$$\theta^T x$$ Compact dot-product form of the linear hypothesis
Vectorization does not change the model itself. It simply makes the notation cleaner, more scalable, and more compatible with linear algebra.
Notation upgrade
The zero feature x₀ = 1 allows the intercept to be included in the same vectorized expression as the other parameters.

Feature Scaling and Mean Normalization

When features have very different scales, gradient descent can become slow and inefficient. Two common preprocessing techniques help fix this: feature scaling and mean normalization.

Feature Scaling

Feature scaling transforms values so that different features have comparable ranges. For example, one feature might range from 0 to 1, while another ranges from 0 to 1000. This imbalance can distort the optimization process.

Preprocessing effect
Combining feature scaling and mean normalization leads to more balanced contours and faster gradient descent convergence.

Mean Normalization

Mean normalization shifts features so that they are centered around zero. This helps gradient descent move more symmetrically in all directions.

$$ x_j := \frac{x_j - \mu_j}{s_j} $$
Symbol Meaning
$$x_j$$ Original feature value
$$\mu_j$$ Mean of feature j
$$s_j$$ Scale (range or standard deviation)

Without Scaling

Features with very different ranges create stretched cost contours, leading to slow and inefficient convergence.

With Scaling + Normalization

Features are more balanced, producing more symmetric contours and faster convergence.

Feature scaling and mean normalisation are simple tricks that help to make GD run much faster and converge in a lot fewer iterations. They are not strictly required for GD to work, but they are very helpful in practice.

Practical Gradient Descent Tips

In practice, optimization is not just about formulas — it is also about checking behavior.

If training is slow
  • Learning rate may be too small
  • Features may need scaling
  • The model may need better feature design
If training is unstable
  • Learning rate may be too large
  • Cost may oscillate or increase
  • Input ranges may be badly unbalanced

Choosing the Learning Rate

In practice, choosing a good learning rate α is one of the most important parts of using gradient descent.

A useful way to evaluate gradient descent is to plot the cost function J(θ) over the number of iterations.

Training behavior
A well-chosen learning rate leads to a smooth decrease in cost. A too-large learning rate may cause divergence, while a too-small one slows convergence.
Good α

The cost decreases smoothly and converges to a minimum.

Too Small α

The cost decreases very slowly, requiring many iterations.

Too Large α

The cost may increase or oscillate, indicating divergence.

A simple debugging rule: if the cost function increases after an iteration, the learning rate is likely too large.

In practice, it is common to try several values of α and observe how the cost evolves. This empirical approach helps identify a good learning rate quickly.

Limits of Straight Lines

A single straight line is not always a good match for real data.

Some relationships are curved or more complex than a basic linear trend. In those cases, a plain line may underfit the data and fail to capture the real pattern.

Model limitation
A straight line may miss the true structure when the relationship between input and output is curved.

Feature Engineering

In many practical problems, model quality depends heavily on how the input features are represented.

Feature engineering means creating new variables from existing ones in order to capture structure that the raw inputs do not expose clearly.

Original Inputs Derived Feature Why It Helps
width, depth area = width × depth Encodes a more meaningful property
x x², x³ Captures non-linear trends
multiple raw variables ratios, products, grouped terms Can reveal hidden relationships
Better representation
New features can make the underlying structure of a problem easier for the model to learn.

Polynomial Regression

One way to model curved patterns is to create polynomial features from the original variable.

Instead of using only x, we can use x, x², x³, and so on. This allows the hypothesis to trace more complex shapes while still using the same linear regression machinery.

$$ h_\theta(x) = \theta_0 + \theta_1 x + \theta_2 x^2 + \theta_3 x^3 $$
Feature Set Model Shape
$$x$$ Straight line
$$x, x^2$$ Parabolic curve
$$x, x^2, x^3$$ More flexible curve
Polynomial regression is still linear in the parameters θ. The non-linearity comes from the transformed features, not from the training method.
Flexible fit
By adding polynomial terms, linear regression can model curved relationships much better than a single straight line.

Normal Equation

Instead of using gradient descent, linear regression can also be solved directly using a closed-form solution known as the normal equation.

This approach computes the optimal parameters in one step, without iterative updates.

$$ \theta = (X^T X)^{-1} X^T y $$

What do these terms mean?

Symbol Meaning
$$X$$ Matrix of input features (each row = one training example)
$$y$$ Vector of target values
$$\theta$$ Vector of parameters we want to learn
$$ (X^T X)^{-1} $$ Matrix inverse (used to solve the system exactly)
The normal equation finds the parameter values that minimize the cost function directly, without needing a learning rate or repeated updates.

Intuition

While gradient descent “walks downhill” step by step, the normal equation jumps directly to the point where the cost function is minimized.

You can think of it as solving a system of equations that describes the best fit for all training examples at once.

Advantages

  • No need to choose a learning rate α
  • No iterative process (no need to monitor convergence)
  • Gives the exact optimal solution (for linear regression)

Limitations

  • Requires computing a matrix inverse, which can be expensive
  • Becomes very slow when the number of features is large
  • Less practical for very large datasets

Gradient Descent vs Normal Equation

Aspect Gradient Descent Normal Equation
Computation Iterative Direct (one-step)
Learning Rate Required Not needed
Scalability Good for large feature sets Slow for large number of features
Feature Scaling Important Not required
Implementation More tuning Simpler for small problems
In practice, gradient descent is preferred for large-scale problems, while the normal equation is useful for smaller datasets or when simplicity is desired.

From Regression to Classification

So far, we have focused on predicting continuous values. But many real-world problems require predicting categories instead.

This type of problem is called classification.

Problem Type Example Output
Regression Predict house price Continuous value (e.g. $250,000)
Classification Spam or not spam Discrete class (0 or 1)
Classification answers: “which category does this belong to?”

Binary Classification

The simplest form of classification is binary classification.

In binary classification, the output can take only two possible values:

$$ y \in \{0, 1\} $$
  • 0 → negative class (e.g. not spam)
  • 1 → positive class (e.g. spam)
Classification task
The model separates data into two categories instead of predicting a continuous value.

Why Linear Regression Fails for Classification

At first, it may seem that we can use linear regression and then apply a threshold:

$$ h_\theta(x) \geq 0.5 \rightarrow y = 1 $$

However, this approach has serious problems.

Problem

  • Predictions can go below 0 or above 1
  • Outliers can drastically change the decision boundary

Result

  • Unstable predictions
  • Poor classification performance
Limitation
A straight line can produce values outside the range [0,1], making it unsuitable for classification.

Towards Logistic Regression

To solve classification problems, we need a model that produces outputs between 0 and 1.

This output can be interpreted as a probability:

$$ 0 \leq h_\theta(x) \leq 1 $$
Logistic regression will allow us to model probabilities instead of unrestricted values.

In the next section, we will introduce the sigmoid function, which transforms linear predictions into valid probabilities.

The Sigmoid Function

To model probabilities, we need a function that always outputs values between 0 and 1.

The sigmoid (or logistic) function transforms any real number into a value in the range [0, 1].

$$ g(z) = \frac{1}{1 + e^{-z}} $$
Sigmoid function
The sigmoid function smoothly maps any input value to a number between 0 and 1.
z → −∞

$$g(z) → 0$$

z = 0

$$g(z) = 0.5$$

z → +∞

$$g(z) → 1$$

Logistic Regression Hypothesis

Logistic regression uses the sigmoid function applied to a linear combination of inputs.

$$ h_\theta(x) = \theta_0 x_0 + \theta_1 x_1 + \cdots + \theta_n x_n = g(\theta^T x) $$
$$ h_\theta(x) = \frac{1}{1 + e^{-\theta^T x}} $$

This means we first compute a linear combination of features, and then pass it through the sigmoid function to obtain a probability.

Logistic regression is still a linear model — the non-linearity comes from the sigmoid function.

Interpreting the Output as a Probability

The output of logistic regression can be interpreted as a probability.

$$ h_\theta(x) = P(y = 1 \mid x; \theta) $$

This means:

  • If hθ(x) ≈ 1 → high probability of class 1
  • If hθ(x) ≈ 0 → high probability of class 0
Logistic regression predicts probabilities, not just labels.

Making a Decision

To turn probabilities into predictions, we apply a threshold:

$$ h_\theta(x) \geq 0.5 \rightarrow y = 1 $$
$$ h_\theta(x) < 0.5 \rightarrow y = 0 $$

This threshold defines the decision boundary that separates the two classes.

Decision boundary
The model separates the input space into regions corresponding to different classes.

Decision Boundary

The decision boundary is the line (or surface) that separates different classes.

It is defined by the set of points where the model is uncertain between the two classes:

$$ h_\theta(x) = 0.5 $$
$$ \theta^T x = 0 $$

This boundary divides the input space into regions where the model predicts class 0 or class 1.

Linear boundary
A straight line separates the two classes when the model is linear.

Linear Decision Boundaries

When we use only the original features, the decision boundary is a straight line.

This works well when the data can be separated linearly.

Good case

Data is roughly separable by a straight line

Bad case

Data has a curved or complex structure

Non-linear Decision Boundaries

By adding polynomial features, we can create non-linear decision boundaries.

For example, using features like x₁² and x₂² allows the model to create circular or curved boundaries.

$$ \theta_0 + \theta_1 x_1 + \theta_2 x_2 + \theta_3 x_1^2 + \theta_4 x_2^2 = 0 $$
Non-linear boundary
Feature transformations allow the model to separate complex patterns.
The model is still linear in θ — the complexity comes from the features.

Key Insight

The shape of the decision boundary depends on the features, not the algorithm.

Better features → better boundaries → better predictions

This connects directly with what we learned earlier: feature engineering and polynomial expansion are powerful tools for increasing model flexibility.

Why Not Use Squared Error?

In linear regression, we used squared error as a cost function. However, this does not work well for logistic regression.

When we apply squared error to logistic regression, the cost function becomes non-convex.

Problem
A non-convex cost function can have multiple local minima, making optimization difficult.
Gradient descent may fail to find the global minimum if the cost function is non-convex.

Logistic Regression Cost Function

Instead of squared error, logistic regression uses a different cost function designed for classification.

For a single training example:

$$ \text{cost}(h_\theta(x), y) = \begin{cases} -\log(h_\theta(x)) & \text{if } y = 1 \\ -\log(1 - h_\theta(x)) & \text{if } y = 0 \end{cases} $$
This cost function heavily penalizes confident but wrong predictions.

Understanding the Cost Function

Logistic cost behavior
Left: when y = 1, the cost decreases as the prediction approaches 1. Right: when y = 0, the cost decreases as the prediction approaches 0.
If y = 1
  • h(x) → 1 → cost ≈ 0 ✅
  • h(x) → 0 → cost → ∞ ❌
If y = 0
  • h(x) → 0 → cost ≈ 0 ✅
  • h(x) → 1 → cost → ∞ ❌

The model is rewarded when predictions are correct and confident, and strongly penalized when they are confidently wrong.

Unified Cost Function

The two cases can be combined into a single expression:

$$ \text{cost}(h_\theta(x), y) = -y \log(h_\theta(x)) - (1 - y)\log(1 - h_\theta(x)) $$

This compact formula works for both y = 0 and y = 1.

This is one of the reasons linear models remain important: with the right features, simple algorithms can become surprisingly powerful.

Cost Function for All Examples

For the entire dataset, we average over all training examples:

$$ J(\theta) = -\frac{1}{m} \sum_{i=1}^{m} \left[ y^{(i)} \log(h_\theta(x^{(i)})) + (1 - y^{(i)}) \log(1 - h_\theta(x^{(i)})) \right] $$
This function is convex, which allows gradient descent to reliably find the global minimum.
One of the simplest debugging tools is to monitor the cost function over time. If it is not decreasing sensibly, something is wrong.

Gradient Descent for Logistic Regression

Logistic regression is trained using gradient descent, just like linear regression.

The update rule looks the same:

$$ \theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta) $$

The key difference is the cost function. Instead of squared error, we now use the logistic cost function.

Same optimization method, different cost function.

Because the cost function is convex, gradient descent will converge to a global minimum.

Putting Everything Together

Despite their differences, linear and logistic regression follow the same core idea.

Component Linear Regression Logistic Regression
Output Continuous value Probability (0–1)
Hypothesis θᵀx g(θᵀx)
Cost Function Squared error Log loss
Task Regression Classification
The structure is the same — only the interpretation changes.

Choosing the Right Approach

Now that we have explored both regression and classification models, we can decide which approach to use depending on the problem.

The choice depends on the type of output, the complexity of the data, and the size of the feature space.

Situation Recommended Approach Why
Predict continuous values Linear Regression Outputs real-valued predictions
Predict categories (0 or 1) Logistic Regression Models probabilities between 0 and 1
Curved or complex patterns Polynomial Features Allows linear models to capture non-linear relationships
Many features (large n) Gradient Descent Scales well with large datasets
Small feature set Normal Equation Provides a direct solution without iteration
Slow or unstable training Feature Scaling + Normalization Improves convergence speed and stability
There is no single “best” method — the right choice depends on the problem, the data, and computational constraints.

In practice, machine learning often involves trying different approaches, observing their behavior, and refining both the model and the features.

Lecture Wrap-up

This lecture introduced two fundamental types of machine learning models: regression and classification.

  • Linear regression predicts continuous values
  • Gradient descent is used to optimize model parameters
  • Feature scaling improves optimization performance
  • Logistic regression models probabilities using the sigmoid function
  • Decision boundaries define how classes are separated
  • The logistic cost function enables stable optimization

These ideas form the foundation for many more advanced machine learning models.

Most machine learning models follow the same pattern: define a hypothesis, define a cost, and optimize the parameters.
