Big Picture

This lecture turns the cost function of linear regression into a trainable procedure. The main tool is gradient descent: start from some parameter values, repeatedly update them to reduce the cost, and stop when the process converges.

Gradient descent is introduced here through linear regression, but the method is much more general and is used in many other machine learning settings.

Gradient Descent Intuition

The lecture presents gradient descent as a hill-descending process: imagine the cost function as a landscape, pick a starting point, look around, and move in the direction that decreases the function most quickly.

repeat until convergence:   θj := θj − α ∂/∂θj J(θ)

The derivative tells you which direction lowers the cost, and the learning rate controls how aggressively you move.

Simultaneous Updates

Incorrect

Update one parameter first, then use that already-updated value while computing the next one in the same iteration.

Correct

Compute every new parameter from the old values first, then apply the full update simultaneously.

This is one of the most important implementation details in basic gradient descent.

Learning Rate

Too Small
Slow training

It moves in the right direction, but progress is inefficient.

Reasonable
Good convergence

The cost goes down efficiently and stabilizes near the minimum.

Too Large
Overshooting

The algorithm may oscillate, overshoot, or fail to converge.

Even with a fixed learning rate, gradient descent can still converge, because the derivative naturally approaches zero near the minimum.

Local vs Global Minima

In general, vanilla gradient descent may end up in different local minima depending on the starting point. But linear regression is a special case: its cost function is convex, so there are no bad local minima to trap the algorithm.

That is why gradient descent behaves especially well on linear regression, provided the learning rate is not too large.

Applying Gradient Descent to Linear Regression

Once the hypothesis and cost function are defined for univariate linear regression, the next step is to substitute those expressions into the generic gradient descent rule.

The lecture also introduces the term batch gradient descent, meaning that each update uses the entire training set.

Multivariate Linear Regression

Univariate

One feature is used to make a prediction.

Multivariate

Multiple features are used together to make a prediction.

hθ(x) = θ0 + θ1x1 + θ2x2 + ... + θnxn

This is the natural extension from a single input feature to many input features.

Vectorized Notation

To simplify notation, the lecture introduces x₀ = 1, so the intercept term can be absorbed into the same vector structure as the other features.

x = [x₀, x₁, ..., xₙ]T,   θ = [θ₀, θ₁, ..., θₙ]T
hθ(x) = θTx

This compact notation makes the multivariate hypothesis and optimization formulas much easier to write and scale.

Feature Scaling

When different features live on very different numerical ranges, the optimization landscape becomes badly stretched, which usually makes gradient descent slower and less stable.

Feature scaling helps make the training process more balanced and efficient.

A major practical lesson from this lecture is that preprocessing is not cosmetic — it strongly affects whether training behaves well at all.

Quick Review Quiz

1. What is the main goal of gradient descent?

To iteratively update parameters in order to reduce a cost function and reach a minimum.

2. What does the derivative tell you in gradient descent?

It tells you the local slope and therefore which direction reduces the cost.

3. Why must parameter updates be simultaneous?

Because every update in a given iteration must be computed from the same old parameter values.

4. What happens if the learning rate is too small?

The algorithm may still converge, but very slowly.

5. What happens if the learning rate is too large?

The algorithm can overshoot the minimum, oscillate, or fail to converge.

6. Why can fixed-learning-rate gradient descent still converge?

Because the derivative gets smaller near the minimum, so the update step shrinks naturally.

7. Why is linear regression special with respect to local minima?

Its cost function is convex, so gradient descent converges to the global optimum.

8. What does multivariate linear regression mean?

Linear regression with more than one feature.

9. Why do we define x₀ = 1?

To absorb the intercept into the same vectorized notation as the other features.

10. Why is feature scaling useful?

Because it makes different features more comparable and helps gradient descent train more efficiently.
