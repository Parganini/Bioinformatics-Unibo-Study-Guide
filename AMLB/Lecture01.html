Big Picture

This lecture frames machine learning as a practical response to problems that are difficult to solve with explicit programming alone. It also sets the tone of the course: not theory for its own sake, but intuition, methodology, and tools that help in real applications.

The course is presented explicitly as an applied ML course. The goal is to learn how to approach data and solve problems, not just memorize formal definitions.

Course Approach and Expectations

Teaching Style
  • Applied and intuition-driven
  • Not a full programming course
  • Python used at an entry level
  • Slides and notebooks provided
Logistics
  • Eight lectures in the basic module
  • Roughly 45 minutes lecture + 15 minutes break
  • Hands-on work with notebooks
  • Material shared during the course

The lecture makes it clear from the beginning that the course is not intended to teach Python from zero, and not intended to be purely theorem-proof-theorem. Instead, it aims to help students build methodological intuition for real ML work.

Core Definitions of Machine Learning

Arthur Samuel
Learning without being explicitly programmed

A foundational early formulation of machine learning.

Tom Mitchell
Performance improves with experience

A machine learns from experience E, on task T, measured by performance P.

The lecture presents both classic definitions and then adds a synthesized one: ML can be seen as building a model and improving it over time by fitting more data under an underlying hypothesis.

Conceptual diagram
A clean restatement of the nested idea from the lecture: ML is part of AI, and Deep Learning is part of ML.

The lecture also argues that the term “AI” is often misleading in practical discussions. In the context of this course, AI is mainly treated as automation of task execution and decision making rather than as general human-like intelligence.

The key idea is not “machines becoming human,” but machines helping perform specific tasks at a scale and speed humans cannot match in practice.

Mitchell’s T / E / P Framework

T
Task

What the system is trying to do.

E
Experience

What the system learns from over time.

P
Performance

How success is measured.

This framework is made concrete with the spam filter example from the lecture: classify emails as spam or not spam, observe user labels, and measure the fraction of correctly classified emails.

Spam Filter Example

T: classify emails as spam or not spam

E: observe how the user labels emails

P: fraction of correctly classified emails

Why it matters

Once you understand T / E / P, you can restate many practical ML problems using the same logic.

Why Now?

Big Data
Massive data availability

Learning becomes more useful when large, rich datasets exist.

Hardware
GPUs and acceleration

Modern compute makes more demanding learning workflows feasible.

Cloud
Accessible large-scale compute

Teams can access infrastructure without owning it directly.

The lecture attributes the recent acceleration of ML to these three factors: the rise of Big Data, technology progress such as GPUs, and the democratization of compute through cloud access.

Types of Machine Learning

Supervised Learning

Learn from data where the correct output is already known.

  • Regression → continuous output
  • Classification → discrete output

Unsupervised Learning

Learn from data without labels and discover structure.

  • group similar items
  • discover hidden structure
  • find patterns in unlabeled data
Reinforcement Learning
Learn from feedback

Improve behavior through rewards, penalties, and repeated interaction over time.

Quick Review Quiz

1. What are T, E, and P in Mitchell’s definition?

T is the task, E is the experience, and P is the performance measure.

A machine learns if performance on the task improves because of experience.

2. What is the task in the spam filter example?

The task is classifying emails as spam or not spam.

3. What counts as experience in the spam filter example?

The system observes how the user labels or flags emails.

4. What is the performance measure in the spam filter example?

The number or fraction of correctly classified emails.

5. How are AI, ML, and Deep Learning related?

ML is a sub-domain of AI, and Deep Learning is a sub-domain of ML.

6. What practical meaning of AI does the lecture emphasize?

Automation of task execution and decision making at scale, rather than general human intelligence.

7. Why has ML adoption accelerated in recent years?

Because of Big Data, hardware progress such as GPUs, and access to large-scale cloud computing.

8. What is the difference between supervised and unsupervised learning?

Supervised learning uses labeled examples, while unsupervised learning discovers structure without labels.

9. Why is house-price prediction a regression problem?

Because the output is a continuous numeric value.

10. Why is tumor diagnosis a classification problem?

Because the output belongs to discrete categories such as benign or malignant.
